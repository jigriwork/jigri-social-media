PHASE 5C — STORIES SYSTEM IMPLEMENTED

1) SCHEMA SUMMARY

- Added migration: supabase/migrations/20260412132000_phase5c_stories_system.sql
- New table: public.stories
  - id (uuid, pk, default gen_random_uuid())
  - user_id (uuid, fk -> public.users.id, on delete cascade)
  - media_url (text, required)
  - media_type (text, constrained to image/video)
  - caption (text, optional)
  - created_at (timestamptz, default now())
  - expires_at (timestamptz, default now() + interval '24 hours')
  - is_active (boolean, default true)
- New table: public.story_views
  - id (uuid, pk, default gen_random_uuid())
  - story_id (uuid, fk -> public.stories.id, on delete cascade)
  - viewer_id (uuid, fk -> public.users.id, on delete cascade)
  - viewed_at (timestamptz, default now())
  - unique(story_id, viewer_id) to prevent duplicate views
- Indexes added:
  - idx_stories_user_id
  - idx_stories_expires_at
  - idx_story_views_story_viewer
- RLS enabled on stories and story_views with policies for:
  - own story creation
  - active/non-expired story visibility (plus own stories)
  - own story deletion
  - own view insertion against active/non-expired stories
  - story view reads scoped to owner/viewer access pattern


2) API SUMMARY

- POST /api/stories
  - Accepts multipart form-data: media + optional caption
  - Uploads media to existing Supabase bucket (posts) under:
    stories/{userId}/{storyId}/media.{ext}
  - Creates story with expires_at = now + 24h

- GET /api/stories/feed
  - Returns active stories grouped by user
  - Marks viewed/unviewed for current user
  - Sorts with followed users first, then others (recent first)

- GET /api/stories/[id]
  - Returns active ordered stories for the provided user id
  - Adds viewed state for current viewer
  - If requested user is current user, includes viewer_count + basic viewer list

- POST /api/stories/[id]/view
  - Records story view (idempotent via upsert + unique constraint)
  - Skips self-view writes

- DELETE /api/stories/[id]
  - Owner-only deletion
  - Removes DB row and attempts storage object cleanup
